# 타임 라인 회고

## 1단계: 프로젝트 기간 동안 중요 이벤트 작성해보기
- 10월 1일: 첫 스프린트 시작
- 10월 1일: 고객사 미팅
- 10월 1일: 상품 목록 페이지 요구사항 분석서 작성
- 10월 8일: 상품 목록 페이지 배포
- 10월 18일: 첫 코드 리뷰 및 리팩터링
- 10월 28일: 프로젝트 회고

## 2단계: 이벤트별 영향력 작성
- 10월 1일: 첫 스프린트 시작
  - 영향: 프로젝트의 전반 방향을 정립, 고객 니즈를 수렴할 수 있는 기틀 마련

- 10월 1일: 상품 목록 페이지 배포, 문서화
  - 영향: 배포와 문서화를 통해 빠르게 결과물을 시각화

- 10월 1일: 상품 목록 페이지 고객사 대상 리뷰, 프로젝트 회고
  - 영향: 리뷰와 회고를 통해 초기 문제점 도출 및 개선 방향 설정

- 10월 8일: 결제 모듈 고객사 대상 리뷰, 프로젝트 회고 및 액션 플랜 도출
  - 영향: 고객 피트백을 통한 구체적인 액션 플랜 도출로 다음 스프린트에 명확한 목표 설정

- 10월 18일: 장바구니 페이지 스토리북 적용
  - 영향: 스토리북 도입으로 UI단위 테스트 및 재사용 수월해짐

- 10월 28일: 리팩터링
  - 영향: 리팩터링을 통해 성능 최적화 및 코드 유지보수성 향상

## 3단계: 개인적 인사이트 도출
- 10월 1일: 첫 스프린트 시작
  - 통찰: 스프린트의 시작시점에 요구사항과 일정이 명확해야 전체 프로젝트 흐름이 안정됨

- 10월 1일: 고객사 미팅
  - 통찰: 고객의 피드백과 요구사항이 프로젝트의 성공에 중요함을 인식

- 10월 1일: 상품 목록 페이지 요구사항 분석서 작성
  - 통찰: 요구사항 분석서를 작성하며 프로젝트의 중요도 이해

- 10월 8일: 상품 목록 페이지 배포
  - 통찰: 배포와 문서화는 고객과의 신뢰를 쌓는데 있어 중요함을 인식

- 10월 18일: 첫 코드 리뷰 및 리팩터링
  - 통찰: 리팩터링 과정이 프로젝트의 품질 및 유지보수에 중요하다는 것을 인식

- 10월 28일: 프로젝트 회고
  - 통찰: 회고를 통해 팀의 성장 포인트 및 개선 필요 영역 확인.

## 4단계: 액션 플랜 수립
- 10월 1일: 첫 스프린트 시작
  - 액션 플랜: 미래 스프린트 계획을 보다 체계적으로 수립하고 명확한 목표 설정

- 10월 1일: 고객사 미팅
  - 액션 플랜: 정기적으로 소통하여 지속적인 피드백 받기

- 10월 1일: 상품 목록 페이지 요구사항 분석서 작성
  - 액션 플랜: 요구사항 분석서를 더 상세하게 작성하여 명확한 지침 제공하기

- 10월 8일: 상품 목록 페이지 배포
  - 액션 플랜: 시간 관리와 효율성을 개선하기 위한 방안 마련하기

- 10월 18일: 첫 코드 리뷰 및 리팩터링
  - 액션 플랜: 정기적인 코드 리뷰 및 리팩터링 세션을 통해 코드의 질을 지속적으로 관리하기

- 10월 28일: 프로젝트 회고
  - 액션 플랜: 완성된 결과물에 대한 회고 문서를 남기며 후속 프로젝트에 반영하기
